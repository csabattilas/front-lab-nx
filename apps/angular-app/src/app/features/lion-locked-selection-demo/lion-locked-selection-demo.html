Lion UI Locked Selection Component

The fl-lion-locked-selection component is a custom web component that extends Lion's LionListbox to implement locked selection functionality, similar to a quiz or test interface.

Key features:

  • Once an option is selected, it becomes locked, preventing further changes
  • Validates answers against a correct answer
  • Provides resolved state for each selection
  • Supports both vertical and horizontal layouts
  • Visually indicates correct answers

Example 1: Vertical Layout (Default)

This example shows the default vertical layout with validation feedback:

  • The answer property is set to "32"
  • The model-value-changed event is used to track changes
  • Visual feedback shows if the answer is correct or incorrect

How much is 17 + 15?

@for(option of ['32','31','33','42']; track $index) { {{ option }} } @if(isSolved() === true) { ✓ Correct answer! } @if(isSolved() === false) { ✗ Wrong answer! Try again. }

Example 2: Horizontal Layout

This example demonstrates the horizontal layout option:

  • The direction property is set to "horizontal"
  • Options are displayed in a row instead of a column
  • Same validation functionality as the vertical layout

@for(option of ['32','31','33','42']; track $index) { {{ option }} } @if(isSolved2() === true) { ✓ Correct answer! } @if(isSolved2() === false) { ✗ Wrong answer! Try again. }

Component API

Properties

Property Type Description
answer String The correct answer value
direction 'vertical'|'horizontal' Layout direction of options (default: 'vertical')
modelValue Array Array of objects with resolved and selectedValue properties

Events

Event Description
model-value-changed Fired when the selection changes

Implementation Details

This component extends the Lion UI listbox component to create a selection that becomes locked after the user makes a choice. It validates the selection against a correct answer and provides visual feedback. The component supports both vertical and horizontal layouts for different UI requirements.
